# Аналитика: воронка «просмотр → корзина → покупка» по открытому e-commerce датасету

**Роль:** бизнес-аналитик / продуктовая аналитика  
**Источник данных (публичный):** [eCommerce events history in electronics store](https://www.kaggle.com/datasets/mkechinov/ecommerce-events-history-in-electronics-store) — автор Michael Kechinov, события реального интернет-магазина электроники (анонимизированы пользователи и сессии).  
**Период данных в наборе:** согласно описанию датасета — **5 месяцев** наблюдений; для воспроизводимости в Kaggle указаны месяцы выгрузки (2019–2020).  
**Дата аналитической записки:** 29.04.2026  
**Инструменты:** Kaggle Notebook (Python + SQL BigQuery-style или DuckDB), визуализация Plotly / Looker Studio

## 1. Бизнес-вопрос

**Вопрос:** В рамках одной пользовательской сессии какова конверсия между этапами **просмотр товара (`view`) → добавление в корзину (`cart`) → покупка (`purchase`)**, и на каком шаге теряется наибольшая доля сессий?

**Почему это важно:** сессионная воронка в e-commerce прямо связана с выручкой; смещение конверсии `cart → purchase` на +0,5 п.п. при миллионах сессий даёт ощутимый прирост GMV.

## 3. Определения метрик (зафиксированы)

| Термин | Определение |
|--------|-------------|
| Сессия | Уникальный `user_session` |
| Сессия с просмотром | В сессии есть ≥1 событие `view` |
| Сессия с корзиной | В сессии есть ≥1 `cart` **после** любого `view` (по времени) |
| Сессия с покупкой | В сессии есть ≥1 `purchase` **после** первого `cart` в этой сессии |
| Конверсия шаг→шаг | Отношение числа сессий на шаге N к числу на шаге N−1 |

*Исключение:* сессии только с `cart` без предшествующего `view` в данных (редкость из-за сбора) — включены в знаменатель «корзина» но помечены флагом `orphan_cart` в контрольной выборке.

## 4. SQL: агрегация воронки по сессиям (DuckDB / аналог)

Ниже запрос рассчитан на таблицу `events(event_time TIMESTAMP, event_type VARCHAR, user_session VARCHAR, …)`.

```sql
-- Шаг 1: признаки сессии
WITH sess AS (
  SELECT
    user_session,
    MAX(CASE WHEN event_type = 'view' THEN 1 ELSE 0 END) AS has_view,
    MAX(CASE WHEN event_type = 'cart' THEN 1 ELSE 0 END) AS has_cart_any,
    MAX(CASE WHEN event_type = 'purchase' THEN 1 ELSE 0 END) AS has_purchase_any,
    MIN(CASE WHEN event_type = 'view' THEN event_time END) AS first_view_at,
    MIN(CASE WHEN event_type = 'cart' THEN event_time END) AS first_cart_at,
    MIN(CASE WHEN event_type = 'purchase' THEN event_time END) AS first_purchase_at
  FROM events
  GROUP BY user_session
),
flags AS (
  SELECT *,
    CASE
      WHEN has_view = 1 AND has_cart_any = 1 AND first_cart_at > first_view_at THEN 1
      WHEN has_view = 0 AND has_cart_any = 1 THEN 1
      ELSE 0
    END AS cart_after_or_without_view,
    CASE
      WHEN has_cart_any = 1 AND has_purchase_any = 1 AND first_purchase_at >= first_cart_at THEN 1
      ELSE 0
    END AS purchase_after_cart
  FROM sess
)
SELECT
  COUNT(*) AS sessions_total,
  SUM(has_view) AS step_view,
  SUM(cart_after_or_without_view) AS step_cart,
  SUM(purchase_after_cart) AS step_purchase
FROM flags
WHERE has_view = 1 OR has_cart_any = 1;
```

**Производная в Excel / Notebook:**  
`cr_view_to_cart = step_cart / step_view`  
`cr_cart_to_purchase = step_purchase / step_cart`

## 7. Выводы (шаблон интерпретации)

1. **Узкое место цепочки** — обычно переход от **корзины к оплате**: даже при относительно высокой мотивации (добавили дорогой товар) пользователи прерывают сессию. Продуктовые гипотезы: гостевой чекаут, прозрачная доставка, доверенные способы оплаты, напоминание об брошенной корзине (с учётом GDPR/согласий).  
2. **Масштаб потерь на верхней воронке** ожидаем для electronics: много сравнительного просмотра без intent купить сегодня — полезно считать **когорты по возвратам в течение 7 дней**, а не только «в одной сессии».  
3. **Рекомендации по данным:** добавить в продукт события `checkout_started`, `payment_failed` (в учебном продукте); в этом датасете их нет — не переинтерпретировать «purchase» как полную картину чекаута.
